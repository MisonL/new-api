## 1. 文档目标

本文件用于定义 `new-api` 从 Electron 桥接到 Tauri 2 期间的最小验证矩阵。

目标不是覆盖所有业务页面，而是锁定桌面迁移最容易失真的主链：

- 新安装
- 重启后会话保持
- 升级后保留数据目录
- OAuth 回调
- SSE
- 托盘与退出行为
- sidecar 故障可观测性

## 2. 预检

执行桥接期验证前，至少确认以下条件：

1. `desktop/tauri-app` 下执行 `bun run smoke` 通过
2. `desktop/tauri-app` 下执行 `bun run check:version` 通过
3. 当前目标平台的 sidecar 已成功生成到 `desktop/tauri-app/src-tauri/binaries/`
4. 应用数据目录为空或已明确区分“新安装验证”和“升级验证”

## 4. 场景执行口径

### 4.1 新安装验证

- 删除桌面应用数据目录
- 启动桌面应用
- 确认首次启动后会生成：
  - `data/new-api.db`
  - `desktop-secrets.json`
- 确认窗口不是在端口刚打开时提前加载，而是在 `/api/status` ready 后显示

### 4.2 重启与升级验证

- 先登录一次，记录页面状态
- 退出桌面应用，再次启动
- 检查登录态是否仍有效
- 升级验证时保留整个应用数据根目录，尤其不能删除：
  - `data/`
  - `desktop-secrets.json`
- 升级后如果出现强制重新登录、历史数据丢失、加密内容异常，直接判定失败

### 4.3 OAuth 回调验证

- 必须用桌面端真实浏览器回调流程验证
- 不接受只在浏览器开发模式下验证通过就视为桌面已通过
- 重点确认：
  - 回调地址仍基于 `localhost`
  - Cookie 未因桌面壳切换而失效
  - 登录成功后能回到桌面主窗口继续操作

### 4.4 SSE 验证

- 选择已知稳定的真实渠道
- 发起至少一次能持续输出的流式请求
- 重点确认：
  - 首包返回正常
  - 中途不会因桌面壳切换出现断流
  - 登录态和同源策略不影响 SSE 持续接收

### 4.5 故障诊断验证

- 人工制造至少两类失败：
  - `3000` 端口在 sidecar 启动前已被占用
  - `3000` 端口被非 New API 的 HTTP 服务占用
  - 数据目录或数据库文件无权限
- 确认桌面端能：
  - 对端口冲突类可恢复错误打开原生服务管理窗口
  - 在服务管理窗口展示当前端口配置和占用进程
  - 修改端口并触发重试启动
  - 对非可恢复错误弹出明确错误标题与摘要
  - 生成 crash log
  - 通过默认系统程序打开日志文件

## 5. 通过标准

以下条件同时满足，才允许认定 Tauri 桥接期主链可继续推进：

1. `bun run smoke` 通过
2. `bun run check:version` 通过
3. 核心验证矩阵 `BRIDGE-01` 到 `BRIDGE-13` 中不依赖真实外部账号或真实渠道的场景全部通过
4. 未出现“升级后数据目录保留但登录态丢失”这类状态面回归
5. 未出现“浏览器开发模式通过，但桌面端 OAuth / SSE 失败”这类假收敛
6. `BRIDGE-06` OAuth 与 `BRIDGE-07` SSE 必须在具备真实外部条件时补充复验记录

## 6. 失败处理

任一核心场景失败时，不得直接宣称“桌面迁移已完成”，必须至少记录：

- 失败场景编号
- 复现步骤
- 当前平台
- 当前版本
- 日志文件路径
- 是否可稳定复现

必要时回退到 Electron 基线继续对照。
