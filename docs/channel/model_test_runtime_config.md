# 渠道模型测试运行配置说明

本文档说明渠道列表中的“测试模型”窗口如何使用运行时配置。该功能只影响本次模型测试请求，不会修改渠道配置。

## 使用场景

模型测试用于确认一个渠道在真实转发链路下是否可用。默认会同时带上渠道已有的运行配置，包括：

- Header Profile 与旧版请求头覆盖
- 参数覆盖
- 请求代理
- 模型重定向
- 测试路径选择

因此，模型测试结果应尽量接近真实客户端调用结果，而不是只做裸连通性探测。

## 默认行为

打开模型测试窗口后，默认端点为：

```text
/v1/responses
```

默认运行配置为：

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| 运行配置 | 开启 | 使用渠道真实运行时配置 |
| 请求头覆盖 | 开启 | 应用 Header Profile、旧版 `header_override`、标签策略等请求头配置 |
| 参数覆盖 | 开启 | 应用 `param_override`，包括 `pass_headers` 透传规则 |
| 请求代理 | 开启 | 使用渠道代理配置 |
| 模型重定向 | 开启 | 使用渠道模型映射 |
| 测试路径 | `/v1/responses` | 控制本次模型测试实际走哪个接口路径 |
| 测试内容 | `hi` | 本次测试发送给模型的最小输入 |
| 最大输出 Tokens | `16` | 控制测试请求输出长度，降低测试成本 |

## 测试路径

模型测试窗口顶部的“测试路径”用于选择本次测试的接口形态：

| 选项 | 说明 |
| --- | --- |
| `/v1/responses` | 默认路径，适合 OpenAI Responses / Codex 类模型 |
| `/v1/chat/completions` | 适合验证上游普通 Chat Completions 模型 |
| `GPT 生图 (/v1/images/generations)` | 适合验证 `gpt-image-*` 生图模型，会把“测试内容”作为 `prompt` |
| `/v1/responses/compact` | 适合验证原生 compact 路径 |
| 其它兼容路径 | 包括 Anthropic、Gemini、Jina Rerank、Embeddings 和自动检测旧逻辑 |

当上游没有开放 compact 路径，但开放了等价的普通 Chat 模型时，可以手动切到 `/v1/chat/completions` 并配合模型映射验证。

## 运行结果怎么看

每个测试结果会附带运行摘要，常见字段含义如下：

| 字段 | 含义 |
| --- | --- |
| 请求头 | 请求头配置是否开启、是否应用、应用了哪些 key |
| 参数 | 参数覆盖是否开启、是否执行 |
| 代理 | 渠道代理是否配置 |
| 模型映射 | 原始模型和最终上游模型是否不同 |
| 路径 | 原始请求路径和最终请求路径 |
| 协议 | 请求协议转换链，例如 `openai_responses -> openai` |

运行配置摘要会固定在一行内横向滚动，避免渠道配置较多时撑高窗口。点击摘要项或高级诊断卡片，会在当前测试窗口上方叠加打开渠道编辑窗口；保存后回到测试窗口可继续测试。

如果失败，窗口会尽量给出诊断分类和处理方向。例如：

- compact 路径不可用：检查上游是否开放 compact 模型，或切换到 Chat 路径并配置模型映射。
- 模型价格缺失：检查模型价格、分组倍率和端点价格配置。
- 请求头失败：检查 Header Profile、请求头名称和值格式。
- 参数覆盖失败：检查 `param_override` JSON、规则模式和 `pass_headers` 配置。
- 上游网关错误：检查上游站点、CDN 超时、Base URL 和代理链路。

## 请求头透传说明

模型测试会把当前 WebUI 请求中的安全请求头复制到内部测试上下文，使 `pass_headers` 能在测试中生效。

以下敏感请求头不会被复制：

- `Authorization`
- `Cookie`
- `New-Api-User`
- `Proxy-Authorization`
- `X-Api-Key`

如果需要验证 Codex CLI、Claude Code 等真实客户端的动态请求头，仍应使用真实客户端发起调用。WebUI 模型测试只能验证当前页面请求里实际存在的非敏感请求头。

## 排障建议

- 常规测试保持运行配置开启。
- 想排查裸连通性时，再关闭请求头、参数覆盖、代理或模型映射中的单项开关。
- 看到 compact 相关失败时，先确认最终请求路径和最终上游模型，再判断是上游未开放路径、模型映射缺失，还是价格配置缺失。
- 不要把模型测试中的路径切换理解为全局自动降级；它只影响本次模型测试。
